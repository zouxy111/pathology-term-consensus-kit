# 病理亚专科术语标准化 SOP

## 定位

本 SOP 用于把“权威术语源 + 历史报告数据 + 主任共识”转成可追溯的术语拍板材料。

适用：

- 胃肠、肺、乳腺、妇科、泌尿等病理亚专科术语统一
- 不同子公司、实验室或主任之间存在历史叫法差异
- 标准化结果后续要进入术语库、模板、规则或搜索映射

不适用：

- 只做一次性文本清洗
- 没有历史报告数据支撑的纯专家讨论
- 直接落地到下游系统但没有投票确认

## 总流程

1. 确定亚专科、主任范围和目标公司。
2. 收集权威术语源，整理成术语底表。
3. 配置历史报告字段映射和目标器官纳排规则。
4. 全量读取所有历史报告，生成扫描日志。
5. 通过 data gate 后，统计术语变体、频次、占比和脱敏例句。
6. 生成证据底稿、主任问卷和会前材料大纲。
7. 主任问卷拍板：同意推荐名 / 建议兼容保留 / 建议弃用 / 需会议讨论。
8. 多数意见 + 总负责人确认；未收敛项进入分歧清单。

## Data Gate

未通过以下检查时，不生成问卷：

- 所有输入文件可打开，所有 CSV/XLSX sheet 可遍历。
- 实际读取行数与 `expected_rows` 一致；未配置时至少记录实际行数。
- 历史报告包含 `company_field` 和 `report_text_field`。
- 目标器官过滤至少接受一部分报告片段。
- 输出材料隐私扫描通过。

## 术语底表

术语底表最小字段：

| 字段 | 含义 |
|---|---|
| `family_id` | 术语家族 ID |
| `category` | 分类，如 WHO 分类、癌前病变、附录 |
| `standard_name` | 推荐标准名 |
| `source_basis` | CAP / WHO / 指南 / 公司规范等依据 |
| `compatible_names` | 历史旧称、兼容名、地方叫法 |
| `deprecated_or_discuss` | 建议弃用或待讨论叫法 |
| `patterns` | 关键词或 `regex:` 正则，用 `|` 分隔 |
| `priority` | `main`、`appendix` 或 `defer` |
| `decision_question` | 主任需要拍板的问题 |

## 历史报告口径

默认口径：

- 公司/实验室字段：实际出报告口径，如 `最终检测子公司`
- 报告文本字段：最终病理结果，如 `单一结果`
- 辅助字段：`诊断`、`送检材料`、`标本类型`

执行原则：

- 不用抽样代替全量扫描。
- 不把非目标器官片段计入目标亚专科。
- 输出材料只保留聚合统计和脱敏短例句。

## 交付物

- `scan_log.csv`：文件、sheet、行数、非空行、digest、错误。
- `evidence.xlsx`：术语摘要、公司频次、变体频次、脱敏例句、data gate。
- `questionnaire.xlsx`：主任拍板问卷。
- `deck_outline.md/html`：会前材料大纲。
- `privacy_report.json`：输出隐私扫描。
- `run_manifest.json`：本次运行的可追溯总账。
